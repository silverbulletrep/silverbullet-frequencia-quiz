## Story

**As a** usuário do quiz funnel,
**I want** transições fluídas e animadas entre as páginas do funil,
**so that** a experiência pareça premium, contínua e profissional — semelhante a um Typeform — eliminando o corte abrupto entre navegações.

---

## Acceptance Criteria

1. Todas as transições de página do funnel são animadas com, no mínimo, um **fade de 200ms**
2. O quiz flow (Steps 1–6) possui **animação direcional** (slide-up para próxima pergunta, slide-down para anterior)
3. **Header/progress bar não apresentam flicker** durante a transição entre páginas onde estão visíveis
4. Transições funcionais em **Chrome, Safari e Firefox** (desktop e mobile)
5. **`prefers-reduced-motion`** é respeitado: degradar para fade rápido (≤100ms) ou sem animação
6. Transição **não adiciona delay perceptível** à navegação em mobile (testado em throttle 3G)
7. **Prefetch** do chunk da próxima página é iniciado ao interagir com o elemento de navegação (`onMouseDown`/`onClick`)
8. Checkout flow (Stripe, PayPal), VSL playback, UTM persistence e health probe **continuam funcionando** sem regressão

## Dev Notes

### Stack & Padrões Relevantes

- **React Router v7.3.0** — Suporta nativamente a prop `viewTransition` em `<Link>` e `<NavLink>`. Ver [React Router View Transitions docs](https://reactrouter.com/en/main/components/link#viewtransition).
  `[Source: package.json — "react-router-dom": "^7.3.0"]`

- **SCSS Modules** — Padrão do projeto. Todos os componentes usam `*.module.scss`. Padrão de `prefers-reduced-motion` já existe em 6 arquivos: `Resultado.module.scss`, `CheckoutSuccess.module.scss`, `Fim.module.scss`, `QuizUI.module.scss`, `DiscountBottomSheet.module.scss`, `CheckoutModal.module.scss`
  `[Source: codebase grep prefers-reduced-motion]`

- **Vite** — Build tool. `React.lazy()` + `Suspense` usado em todas as 27 rotas (App.tsx L8-35). Cada page é um chunk separado via dynamic import.
  `[Source: src/App.tsx]`

### Routing Architecture (App.tsx)

```
BrowserRouter
├── PersistUtmQuery    (L68 — fora do Suspense — UTM persistence)
├── BootHealthProbe    (L69 — fora do Suspense — health check)
├── HeaderAware        (L70 — fora do Suspense — mostra/esconde AuthorityHeader)
└── Suspense (fallback=AppFallback)
    └── Routes
        ├── / → Home
        ├── /quiz → InitialQuestions (EAGER import, não lazy!)
        ├── /age-selection-women → AgeSelectionWomen
        ├── ... (27 rotas)
        └── /recupera → recuoera
```

**Pontos críticos:**
1. `HeaderAware` (L201-207) já está **fora** do `<Suspense>` — NÃO será afetado pela transição
2. `AppFallback` (L44-63) — background `#001736`, texto "Carregando…" — é o fallback genérico
3. `InitialQuestions` é importado **eagerly** (L4) — único page sem lazy import
4. Rotas de hide header: `/checkout-success`, `/recupera` (L203)

### View Transitions API — Compatibilidade

| Browser | Suporte | Notas |
|---------|---------|-------|
| Chrome 111+ | ✅ Full | `document.startViewTransition()` |
| Edge 111+ | ✅ Full | Baseado em Chromium |
| Safari 18+ | ⚠️ Parcial | Experimental, precisa fallback |
| Firefox | ❌ Não suporta | Precisa fallback CSS |

**Decisão arquitetural:** Implementar progressive enhancement:
1. Se `document.startViewTransition` existe → usar API nativa
2. Se não → fallback para CSS animations via `PageTransition` wrapper
