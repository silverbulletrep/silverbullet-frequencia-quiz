## Story

**As a** usuário do quiz funnel,
**I want** que todas as páginas carreguem com o elemento LCP visível o mais rápido possível,
**so that** eu tenha uma experiência de carregamento instantâneo, sem espera perceptível para ver o conteúdo principal da página.

---

## Acceptance Criteria

1. Todas as imagens LCP above-the-fold possuem `fetchpriority="high"` aplicado
2. Nenhuma imagem LCP above-the-fold possui `loading="lazy"` — deve ser `loading="eager"` ou omitido
3. `<link rel="preload" as="image">` adicionado no `index.html` para as imagens LCP das landing pages principais (age-selection-women, age-selection-men)
4. Imagem LCP de `WomenSuccess` comprimida de 401 KiB para **< 100 KiB** (WebP ou AVIF)
5. Imagem LCP de `Resultado` comprimida de 101 KiB para **< 50 KiB** (melhor compressão WebP/AVIF)
6. Todas as imagens LCP possuem atributos `width` e `height` explícitos para evitar CLS (Cumulative Layout Shift)
7. Imagens **não-LCP** abaixo do fold possuem `decoding="async"` aplicado
8. LCP score ≤ 4 em todas as páginas alvo: age-selection-women, age-selection-men, men-success, women-success, transition, resultado, recupera, audio-upsell
9. Checkout flow (Stripe, PayPal), VSL playback, UTM persistence e health probe **continuam funcionando** sem regressão

### Referência de `fetchpriority` Existente

O **único** componente com `fetchpriority="high"` é `InitialQuestions.jsx` (L268, L292), que já segue o padrão correto:
```jsx
<img src={asset('/img/homem.webp')} ... fetchpriority="high" loading="eager" decoding="async" />
```
Usar este mesmo padrão como referência para os demais componentes.

### Preload — Limitações com Vite

- `index.html` atualmente só preloada fontes (L12: Google Fonts Poppins)
- **Problema:** O Vite hasheia nomes de assets no build (`imagem-BaHGqGH5.webp`). Um `<link rel="preload" href="/img/women-18-34.webp">` no `index.html` pode não funcionar após build se o path mudar.
- **Solução recomendada:** Priorizar `fetchpriority="high"` + `loading="eager"` como primary fix. Preload no HTML apenas se os images usam path estável (public folder).
  `[Source: index.html L12, Vite docs]`

### Riscos e Mitigação

- **Risco:** Otimizações de image loading podem causar CLS (layout shift) se `width/height` não estiverem definidos
  - **Mitigação:** Sempre definir `width` e `height` explícitos em imagens LCP (Task 5)
- **Risco:** Preloads no `index.html` podem quebrar após Vite build (nomes hasheados)
  - **Mitigação:** Investigar alternativas (Task 3.3) antes de implementar
