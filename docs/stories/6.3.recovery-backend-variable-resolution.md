## Story

**As a** backend do recovery,
**I want** resolver template, source keys e valores finais a partir de `vw_funnel_lead_compact`,
**so that** o payload enviado ao N8N chegue pronto para a Meta sem if/else por template no workflow.

---

## Acceptance Criteria

1. O backend busca a rota ativa por `message_type + country`.
2. O backend carrega os bindings configurados para a rota/template selecionados.
3. A resolucao de variaveis usa exclusivamente dados de `vw_funnel_lead_compact` neste escopo.
4. O backend suporta `source_key` escalares e subpaths JSONB de `desire`.
5. Quando `resolution_mode = mapped_value`, o backend aplica `value_map` e usa `fallback_value` quando nao houver match.
6. Quando `resolution_mode = pass_through`, o backend usa o valor bruto e aplica `fallback_value` se a `source_key` resolver `null`, vazio ou nao encontrado.
7. Quando `source_key = name` e `resolution_mode = pass_through`, o backend envia somente o primeiro nome com inicial maiuscula e restante minusculo.
8. Se uma variavel obrigatoria continuar sem valor apos a regra de fallback, o backend gera status auditavel e nao envia silenciosamente.
9. O backend nao usa sexo para roteamento estrutural de template.
10. O dispatcher atual continua responsavel pela elegibilidade; esta story adiciona resolucao de template e variaveis, sem reescrever as regras de disparo.

### Name Normalization Rule

Quando `source_key = name` e `resolution_mode = pass_through`:

1. remover espacos extras
2. usar somente o primeiro token do nome
3. converter para `InicialMaiuscula + restanteMinusculo`
4. se nao houver token valido, usar `fallback_value`

### Missing Value Policy

- `mapped_value`:
  - match encontrado -> usa texto mapeado
  - sem match -> usa `fallback_value`
- `pass_through`:
  - valor valido -> usa valor bruto transformado quando necessario
  - valor ausente -> usa `fallback_value`
- se `required = true` e ainda nao houver valor final -> status auditavel e sem envio
